# Code Templates

> Stack-aware code generators for consistent code across your project.
> Claude reads these templates when creating new files to match your project's conventions.

## How It Works

1. `/learn` scans your codebase and auto-generates templates from your actual code
2. `/cook`, `/feature`, `/api` use these templates when creating new files
3. Templates use `{{placeholder}}` syntax for dynamic values

## Template Types

| Template | Used By | Generates |
|----------|---------|----------|
| `api-endpoint.*` | `/api`, `/cook` | New API routes/controllers |
| `component.*` | `/ui`, `/cook` | New UI components |
| `test.*` | `/test`, `/cook` | New test files |
| `service.*` | `/cook`, `/feature` | New service/business logic |
| `model.*` | `/db-migrate`, `/cook` | New database models |
| `hook.*` | `/cook`, `/feature` | New React/Vue hooks |
| `middleware.*` | `/api`, `/cook` | New middleware |

## Custom Templates

Add your own:
```
.claude/templates/
  my-template.ts.hbs     → used when creating files matching pattern
```

Template format:
```typescript
// {{name}} - {{description}}
// Auto-generated by CodePilot

{{imports}}

export {{type}} {{Name}} {{body}}
```

## Integration

Claude MUST check `.claude/templates/` before writing new files.
If a matching template exists, use it as the base structure.
If no template matches, write code normally but consider creating a template for next time.
